## 무엇인가

[galaga-clone](https://github.com/injoon5/galaga-clone)은 Python이랑 pygame으로 짠 갤러그 스타일 세로 슈팅 게임이다. 파일 하나(`galaga.py`), 약 200줄, 스프라이트 PNG, BGM 트랙이 같이 들어있다. 그때 코딩 학원(C3 Coding)에서 게임 개발을 배우고 있었는데 이게 일종의 졸업작품 같은 거였다.

재시작 로직을 좀 손본 `beta/main.py`도 있고, 진행하면서 풀어본 작은 pygame 연습 — 마우스 추적, 폰트 렌더링, BGM 재생 — 들로 가득한 `tutorials/` 폴더도 있다.

## 어떻게 플레이하나

640×650 창, 60 FPS. 화면 아래쪽 우주선(`forces.png`)을 조종하는데, **좌우 방향키**로 움직이고(프레임당 3px, 화면 경계에 고정), **스페이스**로 쏜다.

적(`enemy.png`)은 **1초마다** 화면 위쪽 임의의 X 위치에서 나온다. 각 `Enermy`(그렇다, 코드에 이렇게 철자를 틀려놨다)는 임의의 낙하 속도(2–6)랑 수평 방향을 고른 다음 **아래로 가속**한다(매 프레임 `dy += 0.1`). 좌우 벽에 부딪히면 `dx`를 뒤집어서 튕긴다.

미사일은 스프라이트가 아니라 `pygame.draw.line`으로 그린 **너비 6px짜리 선**이고, 매 프레임마다 **임의의 RGB 색**을 입는다. 아주 2020년스럽다.

적 맞히면 → 점수 +1, 두 객체 다 삭제. 적이 우주선에 닿으면 → 게임 오버 오버레이(`game-over.png`)랑 같이 재시작 버튼이 뜬다. 재시작 누르면 다시 시작.

## 코드 구조

루프 하나에 클래스 넷:

| 클래스 | 역할 |
| ----- | --- |
| `Enermy` | 등장, 이동, 튕김, 미사일과 충돌 |
| `Forces` | 플레이어 우주선 — 이동, 그리기, 발사, 적과 충돌 |
| `Missile` | 프레임당 5px로 위로 날아가고, 화면 밖이면 삭제 |
| `Button` | 재시작 이미지 히트 테스트 |

충돌 처리는 아주 화려하게 수동이다 — 중첩 `while` 루프로 인덱스를 줄여가며 리스트에서 삭제한다. pygame 스프라이트 그룹도 없고 물리 엔진도 없다. 그냥 rect랑 `collidepoint`, 그리고 희망으로 버틴다.

게임 오버 상태는 `gaming = False`로 두고, 적을 다 지우고, 점수를 0으로 리셋한 뒤, `(250, 510)`에 있는 재시작 버튼 rect 위에서의 마우스 클릭을 기다린다.

## 에셋

- `enemy.png`, `forces.png` — 검은색 컬러키 입힌 우주선 스프라이트
- `game-over.png`, `restart.png` — 종료 화면 UI
- `Forget Me Not - Patrick Patrikios.mp3` — 배경 음악 (튜토리얼 폴더에 pygame BGM 로딩 코드가 있다)

## 솔직한 평가

지금 기준으로 보면 엉성하다 — 클래스 이름 오타, while 루프 안에서 리스트 변형, 재시작을 누르기도 전에 게임 오버에서 점수가 리셋되는 것까지. 그래도 처음부터 끝까지 완성한 첫 게임이었고, 적 튕기는 물리를 작동시켰을 때의 그 뿌듯함은 아직도 기억난다. README에는 그냥 `c3coding pygame`이라고만 적혀있고, 결국 흐지부지된 후속작 `codingPro01/galaga`를 가리킨다.
